## 1. 项目总览

### 1.1 项目目标

构建一个能够基于 @objectstack/spec 规范自主迭代开发企业管理软件的AI Agent系统。

**关键成果**:
- ✅ 能理解自然语言业务需求
- ✅ 自动生成符合规范的元数据定义
- ✅ 生成可运行的TypeScript代码
- ✅ 自动测试和质量保证
- ✅ 基于反馈持续优化

---

### 1.2 项目范围

#### 包含范围 (In Scope)
- ✅ 6个核心Agent开发
- ✅ RAG知识库构建
- ✅ 元数据生成引擎
- ✅ 代码生成引擎
- ✅ 测试验证框架
- ✅ 优化迭代机制
- ✅ Web UI界面
- ✅ CLI工具

#### 不包含范围 (Out of Scope)
- ❌ 多租户SaaS平台
- ❌ 应用市场
- ❌ 移动端应用
- ❌ 实时协作编辑

---

### 1.3 技术栈

| 类别 | 技术选型 |
|------|---------|
| **后端框架** | Node.js 20+ / TypeScript 5.3+ |
| **AI模型** | OpenAI GPT-4o, Claude 3.5 Sonnet |
| **向量数据库** | Pinecone / Qdrant |
| **关系数据库** | PostgreSQL 15+ |
| **缓存** | Redis 7+ |
| **前端** | React 18 + Vite + Tailwind CSS |
| **测试** | Vitest + Playwright |
| **CI/CD** | GitHub Actions |
| **监控** | Prometheus + Grafana |

### B. Git工作流

**分支策略**:
- `main`: 生产分支，只接受PR合并
- `develop`: 开发分支，所有功能分支合并到这里
- `feature/*`: 功能分支，开发新功能
- `bugfix/*`: Bug修复分支
- `release/*`: 发布分支

**提交规范**:
```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type类型**:
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具链相关

**示例**:
```
feat(agent): implement requirement parser

- Add entity extraction
- Add relationship detection
- Add validation logic

Closes #123
```
